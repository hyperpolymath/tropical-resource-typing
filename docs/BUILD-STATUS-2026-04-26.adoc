=== F1 — `Tropical_Determinants` line 81

----
*** Failed to apply initial proof method (line 81 of "Tropical_Determinants.thy"):
***  insert x
***   F
*** At command "thus" (line 81)
----

Context (lines 64–88): `sum_tropical_min_mem`, induction by
`finite_ne_induct`, branch where `f x + (∑ y∈F. f y) = f x`. After
`have sum_eq: "(∑ y∈insert x F. f y) = f x"` and
`have "f x ∈ f \` insert x F" by simp`, the line is
`thus ?thesis using sum_eq by auto`.

==== Diagnosis
`sum_eq` rewrites the goal `(∑ y∈insert x F. f y) ∈ f \` insert x F` to
`f x ∈ f \` insert x F`, which is exactly the previous `have` — but
`auto` is not unfolding the rewrite into the membership goal. Likely
fixes:

* `thus ?thesis by (simp add: sum_eq image_iff bexI[OF refl])`
* or: drop `thus` and write
  `from sum_eq show ?thesis by (rule subst[where P="\<lambda>x. x \<in> f \` insert x F"]) (simp add: image_iff)`
* or simplest: `show ?thesis using sum_eq by (auto simp: image_iff)`.

==== Risk
LOW — local rewriting issue; no architectural impact.

=== F4 — `Tropical_Matrices_Full` line 1525 — **ROOT CAUSE**

----
*** exception THM 0 raised (line 308 of "drule.ML"):
***   OF: multiple unifiers
***   ⟦?v ∈ set (butlast ?w); ?v ∈ set (tl ?w); ?no_pos_cycle ?n ?A;
***    ?w ∈ walks ?n ?k ?i ?j⟧
***   ⟹ ∃w'. path_weight ?A ?w ≤ path_weight ?A w' ∧
***           length w' < length ?w ∧ w' ≠ [] ∧
***           hd w' = hd ?w ∧ last w' = last ?w ∧ set w' ⊆ {..<?n}
***   v ∈ set (butlast w)
***   v ∈ set (tl w)
***   no_pos_cycle n A
***   w ∈ walks n k i j
*** At command "using" (line 1525)
----

==== Diagnosis
This is the deep mathematical issue. The lemma
`path_weight_cycle_excise` takes a vertex `v` known to be **both** in
`butlast w` and in `tl w`. From those two memberships you can extract
**two indices** `p, q` with `w!p = v` and `w!q = v`, but in the
absence of an extra constraint they may collapse to the same position
(e.g. `v` appears exactly once at position `p` with `0 < p < length w
- 1`, in which case `v ∈ set (butlast w)` AND `v ∈ set (tl w)` both
hold for the same `p`). The `OF` resolution then has multiple unifiers
because the cycle to excise is not uniquely determined.

==== Two possible fixes

. **Strengthen the lemma signature.** Replace the two membership
  hypotheses with an explicit position pair:
+
----
lemma path_weight_cycle_excise:
  assumes "0 < p" "p < q" "q < length w - 1" "w ! p = w ! q"
  assumes "no_pos_cycle n A" "w ∈ walks n k i j"
  shows "∃w'. path_weight A w ≤ path_weight A w'
              ∧ length w' < length w ∧ w' ≠ []
              ∧ hd w' = hd w ∧ last w' = last w ∧ set w' ⊆ {..<n}"
----
+
The four downstream call sites (838, 1525, 1900, 2044) then need
to prove the position pair from their own pigeonhole/length context
— which they all already have implicitly.
+
PRO: clean semantics; OF has a unique unifier.
CON: each call site needs a small witness construction.

. **Replace the lemma body with a direct pigeonhole on pairs.** Skip
  `butlast`/`tl` membership altogether and quantify over
  `(p, q) ∈ {0..<length w} × {0..<length w}` with `p < q ∧ w!p = w!q`.
+
PRO: removes the `butlast`/`tl` indirection entirely.
CON: bigger refactor.

==== Recommendation
Option (1) — minimal blast radius, preserves the existing call sites'
proof shapes.

==== Risk
HIGH (mathematical, not just syntactic). Estimated 30–60 min of
careful proof writing.

=== F6 — `Tropical_CNO` line 619 / 655

----
*** Bad context for command "qed" (line 655 of "Tropical_CNO.thy")
*** At command "qed" (line 655)
----

==== Diagnosis
Current shape (lines 619–655):

----
proof clarify
  fix i k w assume hi: "i < n" and hw: "w \<in> walks n k i i"
  show "path_weight (trop_mat_star n A) w \<le> (1 :: tropical)"
  proof -
    ... 30 lines ...
    show ?thesis using le_trans[OF step1 step2] .
  qed                          (* line 654: closes inner `proof -` *)
qed                            (* line 655: orphan *)
----

`proof clarify` already discharges the universally-quantified
implications and exposes the witnesses as `fix` in the goal context.
Re-introducing them with a manual `fix i k w assume ...` does not
match the open block, so the inner `show ... proof - ... qed` either
drops the wrong frame or leaves the outer `proof clarify` open with
nothing to close.

==== Fix — pick one

. Drop the manual fix:
+
----
proof clarify
  fix i k w
  assume hi: "i < n" and hw: "w \<in> walks n k i i"
  ... show "path_weight (trop_mat_star n A) w \<le> 1" ...
qed
----
+
i.e. *one* level of indentation, no inner `proof -` / `qed`.

. Or revert to explicit intro:
+
----
proof (intro allI impI ballI)
  fix i assume hi: "i < n"
  fix k w assume hw: "w \<in> walks n k i i"
  show "path_weight (trop_mat_star n A) w \<le> 1"
  proof -
    ...
  qed
qed
----

LOW — pure Isar structure.
