=== Design decisions (do not change without understanding these)

* *Send-latency model*: `send` costs 1, `recv` costs 0. Sender is the
  active party. If extending to symmetric/bidirectional sessions, `grade`
  must be revisited.
* *`recv` written as `val 0 * grade s`*: `val 0` is the multiplicative
  identity, so this is literally `id`. Intentional for structural uniformity.
  The multiplication does no semantic work.
* *`Span` as internal adequacy model*: mirrors grading logic exactly.
  Sufficient to state and prove soundness. Not a full reduction relation.

== Cross-System Connections (not yet formalised)

Three connections exist mathematically but are not yet written as theorems.
Each is a paper section waiting to be written.

=== 1. `soundness` = `max_khop_eq_sum_walks`

Lean `soundness`: static grade = dynamic Span.
Isabelle `max_khop_eq_sum_walks`: matrix power recurrence = sum over walks.

Both say: recursive static definition = inductive operational semantics.
If `Session` is mapped to a `trop_mat` (session type graph = adjacency matrix,
entry node `i`, exit node `j`), Lean `soundness` is a corollary of the
Isabelle theorem. This connection is the bridge for a joint paper.

=== 2. `tropical_grade_le_sequentialTotal` = `tropical_add_idem`

The refinement `max(n1,n2) ≤ n1+n2` is `max a b ≤ a+b` in ℕ, which
follows from `add_idem` in the dioid: tropical addition never overcounts.
State as Isabelle lemma: `trop_add a b ≤ a + b` (as naturals). Trivial, ~2 lines.

=== 3. `spec_par` = `trop_mat_add`

`spec_par s1 s2` with `grade = tAdd (grade s1) (grade s2)` is exactly
`trop_mat_add` on single-edge session matrices. `send`/`recv` are
single-column matrix multiplications. `grade s = trop_mat_star n (session_matrix s) 0 (exit s)`.
Making this explicit gives a machine-checked session type checker backed by
`floyd_warshall`.
